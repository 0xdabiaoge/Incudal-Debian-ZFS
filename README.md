# Incudal Node Setup (Incudal 节点安装管理脚本)

Incudal 节点管理脚本旨在为 [Incudal 面板](https://incudal.com) 打造一款现代化、一键式、高容错的 LXC 容器宿主机部署与维护工具。该脚本完美集成了 Incus 环境配置、Debian 专属的 ZFS 预编译分发以及 RFW 流量拦截防火墙，让你在极短时间内完成业务结点的搭建与销毁。

## ✨ 核心特性

- **🚀 全自动 Incus 部署**：一键完成系统依赖注入、内核参数调整以及网桥管理（支持纯 IPv4 或 NAT+IPv6 模式）。
- **📦 智能 Debian ZFS 处理**：
  - 首选从 GitHub Release ([Incudal-Debian-ZFS](https://github.com/0xdabiaoge/Incudal-Debian-ZFS/releases/tag/Debian-ZFS)) 自动匹配内核拉取预编译模块，实现“秒级挂载”。
  - 智能回退机制：若无适配的预编译包，可提供即时 DKMS 编译选项。编译完成后自动清理 `build-essential` 等工具链。
- **🌐 顶级 IPv6 防冲突智能切割器**：
  - 针对部分宿主机（如DO数字海洋等）下发的极罕见 `/124` 等小段范围 IPv6 资源，脚本能自动进行逻辑切片（拆分为 `/125`），将宿主机自身 IPV6 与LXC网池进行**绝对防冲突**隔离。
- **👻 幽灵双栈同步守护神 (Guardian Daemon)**：
  - 守护神实现宿主机为单一IPV6地址时，会按照 15 秒一轮的节奏进行侦测。面板开一个 IPv4 口子，它就在底层用系统级权限克隆出对应的 `[::]` IPv6 回源隧道，强行从母机 IPv6 赋予纯内网小鸡双栈流量！实现LXC小鸡自由使用宿主机单一的IPV6进行SSH连接和代理入口连接。
- **🛡️ RFW 入站屏蔽防火墙集成**：
  - 基于 Rust && eBPF 的高性能防护。
  - **动态交互配置**：脚本内建界面供您灵活选择屏蔽协议（SMTP, HTTP, SOCKS5, FET[全加密], WireGuard, QUIC）配置。
  - **GeoIP 黑/白名单**：一键锁定国家级来源流量拦截，并支持开启后台 eBPF 拦截日志状态跟踪。
- **🧹 自杀式彻底卸载还原**：
  - 无论系统包损坏或发生了什么样的残留，执行彻底卸载命令能够瞬间强制清理所有的 Incus 后台、存储池、网桥与软件包。

## 🛠️ 环境要求

- **操作系统**: Debian 12、Debian 13、Ubuntu 22.04、Ubuntu 24.04
- **适配架构**: AMD64 (x86_64) 和 ARM64 (aarch64)
- **权限**: 必须以 `root` 用户运行

## 🖥️ 交互式 TUI 菜单

```
(curl -LfsS https://raw.githubusercontent.com/0xdabiaoge/Incudal-Debian-ZFS/main/Incudal.sh -o /usr/local/bin/incudal || wget -q https://raw.githubusercontent.com/0xdabiaoge/Incudal-Debian-ZFS/main/Incudal.sh -O /usr/local/bin/incudal) && chmod +x /usr/local/bin/incudal && incudal
```

**快捷命令：incudal**

**主菜单功能项：**
1. **安装节点 (NAT 模式)**：使用 IPv4 NAT 环境部署 Incus 和 ZFS。
2. **安装节点 (NAT + IPv6 模式)**：部署同时支持 IPv6 栈的 Incus。
3. **安装 RFW 入站流量屏蔽防火墙**：动态定制安装高性能流量防火墙。
4. **查看系统信息**：输出主机的系统概况及目前的 Incus 版本运行架构和 RFW 的运行拦截规则状态。
5. **卸载 RFW**：平滑移除防火墙。
6. **卸载节点**：**【终极杀手剑】** 用于删除清理所有的 LXC 容器、镜像、Incus 设定、ZFS 池、移除相关包以及本安装脚本自身的系统原初状态恢复。
0. **退出**

## ⚡ 部署教程与实操规范 (⭐ 必读 SOP)

以下为各种宿主机部署示例（目前尚未支持宿主机纯IPV6）：

### 示例 A：单IPV6+单IPV6（仅有单公网 IPv4 或加装了一个单 IPv6 的机器）

这类宿主机通常没有自己的连续公网 IPV6 段提供给LXC小鸡独立分发，如需使用此种示例部署请参考如下：

1. **执行安装：** 当本脚本弹出【IPv6 模式询问】面板时，直接选择 `[0] 留空跳过，我只有一个单 IP` 并一直回车即可。
2. **连接面板：** 在 Incudal 网页的节点添加设置中，网络模式**务必且只能选择【IPv4 NAT】**（千万不要选带 `& IPv6` 的任何选项）。
3. **效果：** 面板搭建容器后，是不显示宿主机IPV6地址的，需要自行在节点公告处添加说明，**守护神守护进程**会在脚本部署完毕之后自动运行，只要LXC小鸡添加端口映射，**守护神**会在15秒内将宿主机单一的IPV6地址映射进去，LXC小鸡可以根据节点公告提供的IPV6地址进行SSH连接或者节点搭建做为入口等（IPV6登录SSH端口通常为22或者是自定义映射的LXC小鸡22端口）。

### 示例 B：拥有整段IPV6地址或指定网段IPV6地址（拥有一整段连续分配的 IPv6，如 /64 或者是 /124 碎网段）

这类宿主机拥有整段IPV6地址或指定网段IPV6地址的分配权，提供给LXC小鸡做独立IPV6，如需使用此种示例部署请参考如下：

1. **执行安装：** 当本脚本弹出【IPv6 模式询问】面板时，直接选择带算法推荐的 `[1] (原生整段 /64)` 或是 `[2] (算法切片防冲突提取)`。
2. **计算安全IPV6段：** 此时控制台中央会绿字高亮打印出一个帮你计算好用于填防冲突的 **【安全 IPv6 子网】段**，将其复制后，打开Incudal面板的节点添加处对应位置（IPV6子网段），包括网卡接口名称也会一并计算出来。
3. **连接面板：** 这次要在网络模式中选择 **【IPv4 NAT & IPv6】**。然后在下方的“IPv6 子网”输入框里，将刚才复制的【安全 IPv6 子网】粘贴进去以及网卡接口名称。
4. **效果：** 容器创建部署后，会为每一个LXC小鸡独立分配一个IPV6地址，且不会跟宿主机冲突，也不会与任一LXC小鸡冲突，完全实现独立自主分配。

## ⚠️ 注意事项

- **内核更新锁定**：由于Debian ZFS OOT 原理所限，`Incudal.sh` 在配设成功后会默认利用 `apt-mark hold` 锁死您的内核以防止后台随意升级引发 ZFS 断层挂掉。
